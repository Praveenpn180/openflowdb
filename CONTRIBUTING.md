# Contributing to OpenFlowDB

Thanks for your interest in contributing!

## Getting started

1. Fork the repository on GitHub
2. Clone your fork locally
3. Install dependencies: `pnpm install`
4. Start the dev server: `pnpm dev`
5. Create a branch: `git checkout -b my-feature`

## Before submitting a PR

- Run `pnpm build` and make sure it passes
- Run `pnpm lint` if you changed application code
- Keep changes focused — one feature or fix per pull request
- Update the README if you add user-facing features

## Reporting bugs

Open an [issue](https://github.com/Praveenpn180/openflowdb/issues) with:

- Steps to reproduce
- Expected vs actual behavior
- Browser and OS (if relevant)
- Screenshots for UI issues

## Feature requests

Use the [issue tracker](https://github.com/Praveenpn180/openflowdb/issues) and describe the problem you're trying to solve, not just the solution you have in mind.

## Code style

- TypeScript for application code
- Match existing patterns in the file you're editing
- Prefer small, readable changes over large refactors

## Questions?

Open a [GitHub Discussion](https://github.com/Praveenpn180/openflowdb/discussions) or an issue — we're happy to help.
